MCC VR CCTV Training System

Troubleshooting Guide

Quick Reference for Common Issues

|  |  |  |  |
| --- | --- | --- | --- |
| Issue | Try First | If That Doesn't Work | Escalate |
| VR headset won't connect | Check the link cable is firmly plugged in at both ends | Restart SteamVR from the desktop. Confirm VIBE Hub shows the headset as detected | Contact IT staff |
| EmotiBit sensor LED blinking or off | Remove the sensor and re-strap it on your wrist | Try the spare sensor from the supply shelf | Notify IT staff (sensor may need charging or replacement) |
| Cannot join the session | Close the MCC application and relaunch it | Confirm the supervisor has created a session (check with supervisor) | Contact IT staff to check the configuration |
| No camera feeds visible in VR | Confirm with the supervisor that the scenario has started (you may still be in lobby) | Exit the application and rejoin the session | Contact IT staff |
| Operators can't find the session (supervisor issue) | Confirm the MCC application is running and shows the lobby screen | Ask operators to close and relaunch their application | Contact IT staff to verify network setup |
| Biometric data not showing on dashboard | Check the EmotiBit LED on the operator's wrist (should be solid) | Have the operator remove and re-strap the sensor. Swap for a spare sensor | Contact IT staff |
| Application crashes | Relaunch the application | Check the Saved/Logs folder for crash details. Update GPU drivers and restart the PC | Contact IT staff with the crash log |
| Report won't generate | Confirm the session data was saved (check Data folder) | Click "Generate Report" again | Contact IT staff |

For any issue not listed here, contact IT staff. Do not attempt to modify configuration files or system settings.
