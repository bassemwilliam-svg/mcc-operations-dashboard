# MCCVR Operations Dashboard

Knowledge center for MCC VR training operations. Built with React, TypeScript, and Vite. Runs entirely offline from a browser.

## Quick Start

1. Double-click **Install Dependencies.bat** (first time only)
2. Double-click **Start Dashboard (Dev Server).bat** to run locally
3. Or double-click **Open Dashboard (Offline).bat** to open the pre-built version

## Project Structure

```

├── content/               Editable content (docs, chatbot, data)
│   ├── docs/              Markdown documentation files
│   │   ├── guides/        Software, setup, report, station guides

│   │   ├── scenarios/     Training scenario docs
│   │   ├── training/      Facilitator guides, knowledge transfer
│   │   ├── forms/         Assessment checklists, competency forms

│   └── chatbot/           Dialog tree JSON for the assistant
├── public/                Static assets (images, fonts, favicon)
├── src/                   Application source code
│   ├── components/        Reusable UI components
│   ├── pages/             Page components (one per route)
│   ├── hooks/             React hooks (theme, language, role)
│   ├── lib/               Data and utilities (content, search, i18n)
│   ├── types/             TypeScript type definitions
│   └── styles/            CSS (Tailwind + custom styles)
├── dist/                  Build output (open index.html to run)
└── docs/                  GitHub Pages deployment copy
```

## Config Files

| File | What it does |
|------|-------------|
| `package.json` | Project dependencies and npm scripts. Lists every library the dashboard uses. |
| `package-lock.json` | Locks dependency versions so every install is identical. Auto-generated by npm. Don't edit manually. |
| `vite.config.ts` | Vite build tool config. Sets the base path to `./` for offline use, adds the Tailwind CSS plugin, and defines the `@/` import alias for `src/`. |
| `tsconfig.json` | Root TypeScript config. Just references the two configs below. |
| `tsconfig.app.json` | TypeScript settings for the app code in `src/`. Sets the target, JSX mode, and path aliases. |
| `tsconfig.node.json` | TypeScript settings for config files (vite.config.ts, etc.). Separate from the app config because they run in Node, not the browser. |
| `eslint.config.js` | Linting rules for code quality. Catches common mistakes in TypeScript and React code. |
| `index.html` | The single HTML page that loads the entire app. Vite injects the built JS and CSS here. |

## Batch Files

| File | What it does |
|------|-------------|
| `Install Dependencies.bat` | Runs `npm install`. Do this once after cloning, or after pulling changes. |
| `Start Dashboard (Dev Server).bat` | Starts the Vite dev server with live reload. Opens the browser automatically. |
| `Build Dashboard.bat` | Creates a production build in `dist/`. |
| `Build and Serve.bat` | Builds, then serves the production version locally to test it. |
| `Open Dashboard (Offline).bat` | Opens `dist/index.html` directly in the browser. No server needed. |
| `Clean and Rebuild.bat` | Deletes `node_modules` and `dist`, reinstalls everything, and rebuilds from scratch. |
| `update-content.bat` | Rebuilds after you edit markdown files or chatbot data. Opens the result. |

## Updating Content

All editable content lives in the `content/` folder. To make changes:

1. Edit a `.md` file in `content/docs/` with any text editor
2. Or edit `content/chatbot/dialog-tree.json` for the chatbot
3. Double-click `update-content.bat` to rebuild

The dashboard loads markdown files at build time. Changes won't appear until you rebuild.

## Tech Stack

- **React 19** + **TypeScript** for the UI
- **Vite** for building and dev server
- **Tailwind CSS v4** for styling (dark/light theme)
- **React Router** (HashRouter) for offline-compatible navigation
- **Fuse.js** for client-side fuzzy search
- **react-markdown** + **remark-gfm** for rendering documentation
- **Mermaid** for rendering flowchart diagrams in documents
- **Lucide React** for icons

## Offline Operation

The built dashboard (`dist/` folder) works with zero network access. All fonts, icons, and assets are bundled. Open `dist/index.html` in any browser.
